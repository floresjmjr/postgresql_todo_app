GET /api/todos/{id}

Retrieves todo with id = {id}

Url Parameters

id
The id of the requested todo.

Response

JSON: the todo

1{
2  "id": 1,
3  "title": "todo 1",
4  "day": "11",
5  "month": "11",
6  "year": "2017",
7  "completed": true
8}

Status codes

200
OK - When todo with id = {id} is found.
404
Not Found - When todo is not found. Body: “There is no todo with id = {id}”

POST /api/todos

Saves a new todo.

Request body

Request payload can be either json or a query string

1{
2  "title": "String (Required) : The todo title. At least three characters long.",
3  "month": "String : Format mm. Not saved if format is incorrect",
4  "day": "String : Format dd. Not saved if format is incorrect",
5  "year": "String : Format yyyy. Not saved if format is incorrect",
6  "description": "String : Description of the todo)",
7  "completed": "Boolean : Completion status of todo. Will default to false for new todo"
8}

Response

JSON: The newly created todo with an id attribute

1{
2  "id": 1,
3  "title": "todo 1",
4  "day": "11",
5  "month": "11",
6  "year": "2017",
7  "completed": true
8}

Status codes

201
Created - When the todo is saved
400
Bad Request - When todo cannot be saved (due to incorrect attributes). Body: “Todo cannot be saved”

PUT /api/todos/{id}

Updates a todo. Uses key/value pairs sent via POST to set attributes of the todo. If key/value pair is not present, its previous value is preserved

Request body

Request payload can be either json or a query string

1{
2  "title": "String : The todo title. At least 3 characters long",
3  "month": "String : Format mm. Not saved if format is incorrect",
4  "day": "String : Format dd. Not saved if format is incorrect",
5  "year": "String : Format yyyy. Not saved if format is incorrect",
6  "description": "String : Description of the todo)",
7  "completed": "Boolean : Completion status of todo. Will default to false for new todo"
8}

Response

JSON: The updated todo

Status codes

201
Created - When todo is saved
400
Bad Request - When todo cannot be saved or todo with given id does not exist. Body: “Todo cannot be updated.”

DELETE /api/todos/{id}

Deletes a todo

Status codes

204
No Content - 204 with no body when todo is deleted
400
Bad Request - When todo with id = {id} does not exist. Body: “Todo does not exist”

POST /api/todos/{id}/toggle_completed

Toggles a todo completed. If todo with id = {id} has completed = true, it is set to false and vice versa

Response

JSON: the updated todo

Status codes

201
Created - When todo is deleted
400
Bad Request - When todo with id = {id} cannot be found
